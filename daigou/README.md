# 📦 代购业务项目管理系统

## 🎯 设计理念

### 项目制管理
代购业务按**项目/事件**管理，而不是按月份：
- 每个项目 = 一次完整的代购事件
- 同一系列/同一客户/同一活动的订单 = 一个项目
- 项目可以跨越多个月份
- 项目有独立的生命周期：下单 → 收款 → 发货 → 收货 → 完成

## 💡 利润计算

### 成本计算方法

**Paseli成本汇率**: 统一按 **0.0435 RMB/点** 计算
- 包括RMB充值的Paseli
- 包括历史余额的Paseli
- 简化计算，便于快速核算

**计算公式**:
```
项目成本(RMB) = 使用Paseli点数 × 0.0435
利润 = 收款RMB - 成本RMB
利润率 = 利润 ÷ 成本 × 100%
```

## 🔄 日常操作流程

### 创建新项目

1. **创建项目ID**: `P202512A`（2025年12月第A个项目）

2. **在 projects.csv 添加项目**:
```csv
P202512A,12月XX系列,amnibus,2025-12-01,2025-12-05,0,0,0,待确认,待确认,进行中,
```

3. **创建订单明细文件**: `P202512A_orders.csv`

4. **记录每笔订单**

### 收到客户付款

1. **更新项目订单明细**:
```csv
2025-09-25,2067301,15730,maimai角丸缶バッジ,2026-01下旬,800,2025-11-15,已收款,
```

2. **更新项目总览**:
```csv

```

3. **在个人账本记录**:
```csv
2025-11-15,收入,谷子,22576,RMB,支付宝,amnibus,代购收款-P202509A
```

### 项目完成归档

1. 确认所有订单收款
2. 确认客户收货无问题
3. 计算项目总利润
4. 更新状态为"已完成"
5. 可选：移到 `archive/` 文件夹

## 📝 备注

### 项目制的优势

1. ✅ **清晰的项目边界**: 同一活动/系列订单统一管理
2. ✅ **完整的生命周期**: 从下单到收货全程追踪
3. ✅ **独立的利润核算**: 每个项目独立计算盈亏
4. ✅ **便于对账**: 项目级别对账更清晰
5. ✅ **历史可追溯**: 归档后仍可查询
6. ✅ **跨月项目支持**: 可以跨越多个月份统一管理

### 项目可以跨月的典型案例

**P202509A项目** 就是典型的跨月项目：
- 9月25日开始下单
- 10月继续追加订单
- 11月4日最后一单
- 2026年1-2月分批发货
- 整个项目生命周期跨越5个月

这正是项目制的优势：按照业务事件的自然边界来组织，而不是被日历月份割裂。

### 与月度账本的区别

**个人账本**（按月份）:
- 记录现金流（钱的流动）
- 按时间顺序记录
- 用于个人财务管理
- 文件按月份分割（9.csv, 10.csv, 11.csv）

**代购账本**（按项目）:
- 记录业务流（订单状态）
- 按项目/事件组织
- 用于代购业务管理
- 一个项目可能涉及多个月份的个人账本
